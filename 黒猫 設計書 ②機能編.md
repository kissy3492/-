# 1. ウィンドウ配置

═══════════════════════════════════════════════════════════════════

ウィンドウをパレットから選んだ枠（左半分/右半分/四分割）へ整列し最前面固定する
対話モード。コードネーム廃止、機能名「ウィンドウ配置」。

## 1.1 目的

作業のたびに窓が隠れる/前面に出したいものが奥へ行くを解消。クリック→枠選択
だけで整然と並べ、選んだものを最前面固定して「常に見えていてほしい」を満たす。

## 1.2 確定仕様

- 対象モニタはモード起動時に毎回選ぶ（対象モニタ=配置先＝モニタ間移動も可）。
- 6枠を毎回パレットから選択。占有済み枠は自動で選択不可。
- 同時固定は最大4枠（四分割のセル数）。重なり構造的にゼロ。
- 解除は三系統併用（再選択トグル/全解除/個別解除）。
- モード継続型（1枚置いても続く・Escで終了）。配置先UIはパレット。
- すき間は外周マージン＋枠間ガター（既定各8px）。

## 1.3 設計の心臓 ―― セル占有モデル

画面を四分割した4セル（LT/LB/RT/RB）の組合せで6枠を表現する。

|枠                                    |セル集合   |
|-------------------------------------|-------|
|左半分                                  |{LT,LB}|
|左上 {LT} / 左下 {LB} / 右上 {RT} / 右下 {RB}|       |

**枠が選べる⇔そのセル集合が現占有集合と互いに素**。この単一規則から、要望が
すべて自動的に導かれる:

- 「左半分を選んだら左半分/左上/左下が消える」← {LT,LB}占有で交わるため
- 「最大4枠」← セルが4つ
- 「絶対重ならない」← 1セル1ウィンドウ

> 相乗効果: 重ならないので、複数を最前面固定しても前面の奪い合いが起きない。
> 「重ならない設計」が「最前面固定同士の競合」まで同時に解決する。

占有集合の更新: 配置でセル追加、解除/ウィンドウ消滅でセル解放（死活監視）。

## 1.4 配置矩形（ギャップつき）

作業領域(WX,WY,W,H)から外周マージンを引いた使用可能領域を縦横ガターで二分割。
端数は右/下へ寄せ（はみ出すより1px痩せる側に倒す＝重なり防止優先）。

|枠  |x / y / w / h                           |
|---|----------------------------------------|
|左上 |leftX, topY, colW, rowH ／ 左下 leftX, botY|
|右上 |rightX, topY ／ 右下 rightX, botY          |
|左半分|leftX, UY, colW, UH ／ 右半分 rightX, UY    |

## 1.5 操作フロー（モード継続型）

起動→対象モニタ選択（単一はスキップ/複数はオーバーレイで番号選択）→
インジケータ表示→ウィンドウをクリック（未ピン=パレット表示→枠選択→復元→
配置→最前面固定→占有更新／ピン済み=トグル解除）→繰り返し→Escで終了
（ピンは維持・持続）。

クリック対象は WindowFromPoint→ルート（GA_ROOT）。自プロセス・デスクトップ・
タスクバーは除外。

## 1.6 パレットUI

ウィンドウクリック直後、画面位置と一致する見取り図状の6ボタン（左半分バー｜
2×2象限｜右半分バー）。占有枠はグレーアウト＋無効。全占有なら「空き枠なし」。
パレット外/Escで当該配置キャンセル（モード継続）。

## 1.7 最前面固定と解除（三系統）

- 固定: SetWindowPos HWND_TOPMOST。非重なり保証で奪い合い無し。
- 解除1: 配置モード中にピン済みをクリック→即解除（パレット出さない）。移動は
  解除→選び直しの2手（予測可能性優先）。
- 解除2: トレイ「全固定解除」。
- 解除3: トレイ「固定中ウィンドウ…」一覧から個別解除。

## 1.8 Win32 詳細

最大化なら SW_RESTORE→SetWindowPos（位置・サイズ＋TOPMOST）→applied_rect 実測
記録。Per-Monitor DPI Aware v2（①§B12）。昇格ウィンドウ等は移動不可→通知し
ピンしない。

## 1.9 データモデル（ピンレジストリ・プロセス内のみ）

hwnd / title / monitor_id / zone / cells / applied_rect / was_topmost /
was_maximized。対象モニタの占有集合 = 全項目cellsの和集合。死活監視（既定2秒
IsWindow）で消滅窓のセル自動解放。

## 1.10 エッジケース

操作不能窓→通知しピンせず / 最小サイズ超過（枠に収まらない）→通知し(a)そのまま
(b)取消 を選択（OSが縮小拒否する実寸までは保証外・割り当て矩形の非重なりは保つ）
/ モニタ構成変化→安全終了しやり直し案内 / 手動移動→占有は自動解放しない。

## 1.11 設定

margin_px(8) / gutter_px(8) / hotkey_placement(Ctrl+Alt+W) / topmost_on_place
(true) / restore_on_release(false) / recording_conflict(pause) /
watch_interval_sec(2) / palette_follow_cursor(true) / monitor_picker_keys(true)。

## 1.12 連携

- 呼び出し→配置・作業状態→配置 の委譲を受ける（対象＋枠を受け取り矩形計算/
  占有/topmostを担う・③）。
- コマンド供給: 配置開始/全固定解除/固定中一覧。

## 1.13 チェックリスト

- [ ] 対象モニタ選択（単一スキップ/複数オーバーレイ）
- [ ] セル⇔枠対応・選択可否＝互いに素・占有の追加/解放（§1.3）
- [ ] 不変条件: 非重なり・最大4枠（§1.3）
- [ ] 外周マージン＋ガター矩形・端数は痩せる側（§1.4）
- [ ] パレット（位置一致・占有グレーアウト・Escキャンセル）（§1.6）
- [ ] 三系統解除（§1.7）／モード継続・Esc終了・ピン維持（§1.5）
- [ ] 最大化解除→SetWindowPos→実測・移動不能通知・最小超過確認（§1.8,§1.10）
- [ ] 死活監視でセル自動解放（§1.9）

# 2. ウィンドウ呼び出し

═══════════════════════════════════════════════════════════════════

特定アプリ/ウィンドウをホットキー一発で前面化（無ければ起動）。配置と対で
「呼び出して並べる」を成立させる。

## 2.1 目的・確定仕様

目的: 目的の窓を毎回探す手間を消し、必ず前面に出す（無ければ起動）。
確定: 在れば前面化/無ければ起動の二段 / スロット制（exe・タイトル条件・起動
パス）/ スロット直割当キー＋呼び出しパレット / 前面化↔復帰トグル / 配置連携 /
複数該当は直近・一致で選別、曖昧なら選択 / PowerShell非依存起動。

## 2.2 解決ロジック（出現の保証）

条件に合う既存窓を探す→在る(1つ)=前面化／在る(複数)=タイトル条件→直近→選択／
無い=登録パスから起動。これにより「押せば必ず出る」。

- 前面化はOSのフォーカス強奪制限に配慮（スレッドアタッチ等・①§B12）。阻まれた
  らタスクバー点滅で代替。
- 復帰トグル: 前面化直後に同キー再押下で呼出前の窓へ戻る（時間窓既定3秒）。
- 起動後は出現待ち（タイムアウト・既定8s）→配置連携へ。出現検知失敗は起動のみ。

## 2.3 スロット登録

マッチ条件（exe・タイトル部分一致・クラス＝AND）。「今フォアグラウンドの窓を
登録」ワンボタンで雛形自動取得。起動パス（exe直/.lnk・プロセスパス自動補完）。
名前・アイコン（exe由来）・割当キー・配置枠（任意）。

## 2.4 呼び出しパレット

Ctrl+Alt+Space で全スロット一覧（名前・アイコン・割当キー・状態=起動中/未起動）。
数字/↑↓+Enter、インクリメンタル検索。配置枠紐付きは枠アイコン表示。

## 2.5 配置連携（対の完成）

スロットに配置枠を紐付け→呼び出し後その枠へ配置（配置の矩形/占有/topmost使用・
③）。「作業開始」キーで複数スロットを順に呼んで並べる簡易レイアウト（§2.5.2）。
連携はシェルのモジュール間IF越し（疎結合）。

## 2.6 永続化・設定・エッジ・チェック

永続: スロット定義（条件・パス・名前・割当・枠）。機微なし。
設定: hotkey_palette(Ctrl+Alt+Space) / slot_hotkeys(Ctrl+Alt+1〜9) /
toggle_back_sec(3) / launch_if_absent(true) / appear_timeout_ms(8000) /
multi_match(recent_or_pick) / quickstart_slots([])。
エッジ: パス不正→前面化のみ / 強奪阻止→点滅 / 起動後窓出ず→次回へ / 複数該当→
候補リスト / 多重起動回避アプリ→前面化のみ設定。
チェック: [ ] 二段解決 [ ] 複数選別 [ ] 復帰トグル [ ] 出現待ち [ ] スロット
登録/自動補完 [ ] PowerShell非依存起動 [ ] 配置枠連携 [ ] 永続化。

# 3. 作業状態 保存・復元

═══════════════════════════════════════════════════════════════════

開いていたファイル群とその配置を保存し、翌日以降に同じ作業環境を呼び戻す。
呼び出し＋配置の基盤を再利用する束ね機能。

## 3.1 目的・確定仕様

目的: 複数ファイルを毎回閉じて開き直す手間を消す。
確定: 保存対象は**ユーザーのアプリ＋ファイル＋配置**（黒猫自身の機能状態は対象外）
/ ドキュメント系=アプリ＋パスで正確復元・ファイル無し窓=アプリ起動のみ＋枠配置 /
1スロット上書き / 手動のみ / 機微（転記値・履歴等）は保存しない。

## 3.2 設計思想

- 「収集して記述する」「記述から呼出（起動/前面化）と配置（並べる）を駆動する」
  橋渡しに徹する（新ウィンドウ操作を持たない）。
- 黒猫自身は保存しない（二重管理と機微残留を避ける明確な線引き）。
- 復元はベストエフォート＋正直なレポート。既存窓を勝手に閉じない（非破壊）。

## 3.3 収集（保存時）

可視トップレベル窓を列挙（黒猫自身・デスクトップ・タスクバー除外）。各窓を記述化:
exe/プロセスパス・タイトル・クラス・**ファイルパス特定**・配置記述・最小最大。

ファイルパス特定（多段・確実な順）: ①アプリ固有手段（Office実行中ドキュメント
情報等）②タイトル推定＋既定/最近フォルダ突合（弱い補助）③不明なら「アプリのみ」。

> 不確実なパスで誤ファイルを開くより「アプリのみ」に落とす（誤オープン回避）。

## 3.4 復元

- ドキュメント系: アプリ＋パスで開く（関連付け/引数起動）。既開なら前面化。
  PowerShell非依存。
- ファイル無し窓: アプリ起動のみ＋枠配置（ブラウザ/業務システムの特定画面は
  戻せず手戻し前提・レポートで明示）。
- 既存窓を閉じない（非破壊）。出現待ち（タイムアウト）→配置。失敗はスキップ。

## 3.5 配置の記録・復元

保存時、実矩形が6枠に近ければ枠として記録（再現安定）、合わなければ実矩形
（モニタ相対）。モニタ識別子も記録。復元は枠→配置基盤で再計算、実矩形→モニタ
相対（画面外は可視領域へ寄せ）。**復元時の最前面固定は既定オフ（並べるだけ）**。
配置失敗してもオープンは成功扱い（開ければ目的の大半は達成）。

モニタ構成変化: 該当モニタ無→主モニタへ寄せ再計算、解像度違→枠は再計算/実矩形は
クランプ。ずれの可能性をレポートに添える。

## 3.6 復元レポート（正直さ）

完全復元 / アプリのみ / パス不明 / 起動失敗 / 配置のみ失敗 を一覧表示。「何が
戻り何を手で補うか」を一目で。レポートは保存・ログしない。

## 3.7 潔白性

保存するのはファイルパス・アプリパス・窓情報・配置のみ。中身・履歴・転記値・
付箋本文・スクショは保存しない。パスが事案名等を含み機微になりうる点に配慮:
ローカル限定・ログにパスを書かない・「今すぐ消去」。

## 3.8 データモデル

スロット: saved_at / monitor_profile / entries。エントリ: exe/process_path /
title/class / file_path / doc_kind / monitor_id / zone / rect / min_max / order。
上書き時1世代バックアップ。機微データは一切持たない。

## 3.9 復元の深さ（境界の明示）

できる: 同アプリで同ファイルを開き同じ枠に並べる。できない: ファイル内の
スクロール位置・カーソル・未保存編集（OS/アプリへ外部介入不可の原理的制約）。
本機能は「開き直しの手間をなくす」までを確実に担う。

## 3.10 設定・チェック

設定: hotkey_save(Ctrl+Alt+S) / hotkey_restore(Ctrl+Alt+R) /
confirm_before_save(true) / confirm_before_restore(true) /
topmost_on_restore(false) / skip_if_already_open(true) / appear_timeout_ms(8000)
/ keep_backup(true) / exclude_exes([])。
チェック: [ ] 収集（除外・記述化・パス多段特定・0件は保存せず）[ ] 復元（ドキュ
=パス/ファイル無=アプリのみ・非破壊・出現待ち）[ ] 配置（枠推定/実矩形・既定
topmostオフ・モニタ変化・配置失敗でも成功扱い）[ ] レポート [ ] 機微非保存。

## 3.11 連携

呼出（起動/前面化）・配置（枠）へ委譲。第1波で複数スロット化（事案ごと・③）。
コマンド供給: 保存/復元（一覧は動的）。

# 4. ファイル検索

═══════════════════════════════════════════════════════════════════

登録した調査フォルダ群をファイル名で串刺し検索しEnterで開く。「探す」を担い、
呼出（出す）・作業状態（戻す）と「ファイルへの到達」3経路を完成させる。

## 4.1 目的・確定仕様

目的: 大量の資料から目的ファイルを毎回掘る手間を消す。
確定: ハイブリッドindex（起動時一括→差分監視＋定期再走査）/ **ファイル名のみ**
（中身検索しない＝潔白性・軽量）/ 登録フォルダ串刺し（ドライブ全体しない）/
Enter=既定アプリ・Shift+Enter=フォルダ・Ctrl+C=パスコピー / 共有フォルダ配慮 /
拡張子・更新日時・フォルダで絞り込み / 索引はメタのみ。

## 4.2 設計思想

- 「開く導線」に徹し、ファイル操作（移動・削除）はしない。
- 串刺しは登録フォルダ限定（速度＋潔白性＝無関係領域を見ない）。
- **ファイル名のみ＝潔白性の核**: 中身・サムネ・プレビューを一切持たず、名前・
  パス・更新日時・サイズのメタのみ。調査資料の中身をローカルに溜める懸念を構造
  排除。

## 4.3 インデックス（ハイブリッド）

①起動時/登録時に一括走査→索引化 ②以降は差分監視（フォルダ変更通知）で増減反映
③定期再走査（既定30分）で取りこぼし補正。索引情報＝名前/フルパス/拡張子/更新
日時/サイズ/所属フォルダ（中身なし）。

共有フォルダ配慮: 分割走査でUIを固めない・切断時は一時スキップ→復帰で再走査・
通知不安定なら定期再走査主体・「重い」マークで進捗表示。索引はローカル永続
（次回起動を速く・メタのみ）。既定除外（~$/一時/隠し）。索引上限・警告。

## 4.4 検索

照合: ファイル名（名前部＋拡張子）へ 部分一致・fuzzy・頭文字・単語境界一致。
日本語はファイル名への部分一致が主（読み変換しない＝確実性）。
スコア: 一致良さ＋学習（最近/よく開く）＋更新新しさ（弱め）。
絞り込み: 拡張子・更新日時（今日/7日/30日/任意）・対象フォルダ。簡易構文
`集計 ext:xlsx` `照会 days:7`（設定）。同名異所はフォルダで区別表示・一致箇所
ハイライト。

## 4.5 開く

Enter=既定アプリ（関連付け起動・PowerShell非依存・既開は前面化）／Shift+Enter=
選択状態でエクスプローラー／Ctrl+C=フルパスをクリップボード（履歴に乗る・転記へ
送る導線も）。開いたらパネルを閉じアプリへフォーカス。

## 4.6 学習・潔白性

学習: 開いた回数・最近性でスコア押上げ。無入力時に「最近/よく開く」上位。永続は
パスごとの開回数・最終時刻のみ（クエリ文字列は保存しない）。
潔白性: 中身を読まない・索引/学習はローカル限定・クエリ非保存・パスをログに
書かない・「インデックスを今すぐ消去」。

## 4.7 データモデル・設定・エッジ・チェック

対象フォルダ: path/recursive/kind(local|shared)/enabled/excludes。索引項目:
name/path/ext/mtime/size/root。学習: path/open_count/last_opened。
設定（名前空間 `search.` 配下・①§B10。`hotkey_panel`等のキー名は他機能の同名
設定と名前空間で分離され衝突しない）: hotkey_panel(Ctrl+Alt+F) / target_folders([]) / rescan_interval_min(30) /
watch_enabled(true) / default_excludes(~$*,*.tmp,隠し) / max_index_files(200000)
/ query_syntax(true) / skip_if_already_open(true) / persist_index(true)。
エッジ: フォルダ消失/権限なし→スキップ通知 / 共有切断→一時スキップ→再走査 /
索引未完→既存で検索＋「更新中」/ 索引後に移動削除→開く時に不在検知し除去。
チェック: [ ] 対象登録（再帰/種別/除外）[ ] ハイブリッド索引＋共有配慮 [ ] 索引
メタのみ・ローカル永続 [ ] あいまい照合＋絞り込み [ ] 学習（頻度のみ）[ ] 開く
3系統 [ ] 中身読まない・クエリ非保存。

## 4.8 連携

パスコピー→履歴 / パス→転記 / 検索結果→作業状態へ束ね（③）。コマンド供給:
検索/再インデックス/インデックス消去。

# 5. 付箋ピン

画面の隅に小さなメモを最前面で貼る。配置のtopmost基盤を流用。刹那メモに特化し
Continuum Commander（腰を据えたタスク管理）と棲み分ける。

サイズ・色・折畳・最前面・所在モニタ。画面外付箋は起動時に可視領域へ寄せる。

# 6. 領域キャプチャ

═══════════════════════════════════════════════════════════════════

画面の一部を矩形で切り出し最前面の小窓に固定。付箋の画像版。OCRは使わない。

## 6.1 目的・確定仕様

目的: 別資料の表や数値を見ながら手元で作業（元窓を探さず見たい所だけ手元に）。
確定: 既定は静止画スナップ・設定でライブ複製も可 / 複数同時・拡大縮小 / 画像/
クリップボード任意保存・窓は揮発（閉じたら消える・再起動で残さない）/ 全退避/
全消去ホットキー / 付箋topmost＋スクショ基盤流用・OCRなし。

## 6.2 設計思想

- 付箋の画像版応用（topmost小窓・移動/リサイズ/全退避/束ね/一覧をほぼ流用、
  本体をテキスト→画像へ）。配置のモニタ/DPIも共有。
- 静止が基本・ライブは選択（既定ライブにしない＝再キャプチャ負荷と意図せぬ機微の
  動的取り込みを避ける）。
- **OCRは使わない**（けいとが断念した経緯を踏まえ画像のまま扱う）。
- 機微度は付箋より高い前提→**揮発**（永続化しない）＋全消去手段必備。

## 6.3 領域選択・キャプチャ窓

選択: 全画面（マルチモニタ全体）に半透明オーバーレイ→ドラッグで矩形（寸法px
表示）→離して確定。極小は無効。物理ピクセル取得でボケ低減。取得画像はメモリ
保持（ディスクに書かない）。
窓: 静止スナップ（既定・切出し瞬間を固定）／ライブ複製（ソース矩形を既定500ms
再キャプチャ・退避/最小化中は停止・元喪失は直近フレーム維持）。窓ごとに切替可。
細いドラッグ帯＋画像＋角ハンドル・折りたたみ。既定 HWND_TOPMOST・個別トグル。

## 6.4 複数・拡大・保存・機微対策

複数同時（ソフト上限警告・重ならせない）。ホイール拡大縮小（実寸/フィット/倍率）。
一覧・束ね・色枠。保存: 画像コピー(Ctrl+C・履歴に乗る)／PNG保存（明示操作時のみ・
自動保存しない）。
**機微対策（中核）**: Ctrl+Alt+Shift+G 全退避（来客・画面共有時に即クリア）／
トレイから全消去（窓と画像をメモリ破棄）／永続化しない（再起動で復活させない＝
機微画像の長期残留を構造防止）／画像をログに残さない・外部送信ゼロ。

## 6.5 設定・エッジ・チェック・連携

設定: hotkey_capture(Ctrl+Alt+G) / hotkey_hide_all(Ctrl+Alt+Shift+G) /
default_mode(snapshot) / live_interval_ms(500) / topmost_default(true) /
fullscreen_behavior(retain) / zoom_default(fit) / soft_limit(12) / save_dir。
エッジ: キャプチャ失敗→中止通知 / 極小→やり直し / ライブ元隠れ→直近維持 /
多数で重い→上限警告・束ね/全消去 / モニタ変化→可視領域へ寄せ / DRM保護→黒く
写る制約。
チェック: [ ] オーバーレイ選択・物理px・メモリ保持 [ ] 静止/ライブ（停止/直近
維持）[ ] 既定topmost・個別トグル [ ] 複数・拡大・一覧/束ね [ ] 画像コピー(履歴)
・明示保存のみ [ ] 全退避/全消去/永続化しない/ログに残さない。
連携: 画像→履歴（クリップボード経由・揮発）、画像→付箋（将来・③）。コマンド
供給: 領域選択/全退避/全消去。

# 7. 項目転記

═══════════════════════════════════════════════════════════════════

読み取り元から複数項目をコピーで集め、書き込み先（自分のExcel/Word）へTab送り/
TSV貼付で一括流し込む。

## 7.1 目的・確定仕様

目的: システムから自分の資料へ毎回コピペを繰り返す手間を、1〜2アクションに圧縮。
確定: 読み取り=クリップボード経由（ドラッグ選択→自動コピー→蓄積）/ 書込=自動
入力（自分のOffice等）/ 複数項目まとめて / 表はTSV一括 / マッピング保存しない
（③第1波で追加）/ 単純コピー（整形は③第1波）/ ホットキー2つ / 先頭欄クリック→
Tab送り連続入力 / 流し込み前に確認プレビュー。

## 7.2 設計思想

- **2フェーズ（収集→流し込み）**: 集めると流すを明確分離。収集中に書込先へ触れ
  ても誤爆せず、流す直前に確認できる。
- クリップボード経由＝画面素性に依存しない堅い実装（ドラッグ選択＋自動コピー）。
- **非対称な安全設計（読み元は読むだけ）**: 読み元には Ctrl+C のみ・自動書込
  しない（職場システムを自動操作しない＝RPA的挙動を読み元に持たない）。書込先は
  自分のOfficeに限定し自動入力を許可。「基幹システムを自動操作する」懸念を構造
  排除。
- クリップボードは借りたら返す（退避・復元・①§B12）。

## 7.3 収集フェーズ

開始（ホットキー1/トレイ）→インジケータ＋収集パネル→クリップボード退避。
1項目: ドラッグ選択完了（ボタン解放）検知→自動 Ctrl+C→クリップボード読取→
バッファ末尾へ→パネルに追加。
確実化: Ctrl+C後にクリップボード更新をポーリング（取れなければ「コピーできません
でした」）。連続重複は低強調。
誤爆防止: 自プロセス上の選択は対象外・極小ドラッグはコピーしない。

## 7.4 収集パネル

番号・内容プレビュー・種別（テキスト/表TSV）。並べ替え（流し込み順に直結）・
削除・手動編集（誤選択の手直し）・全クリア。流し込み前に順番と中身を目視確認
できる場（§7.6の確認と二重の誤入力防止）。

## 7.5 流し込みフェーズ（自動入力）

確認プレビュー（即入力せず「N項目を現在フォーカスの欄から順に入力します」＋項目
一覧＋書込先ウィンドウ名表示＝意図しないアプリへの事故を気づける）→実行。
Tab送り: バッファ先頭から順に入力、最終以外は項目間に Tab（横方向・既定）。設定で
Enter（縦方向）。表項目はTSV展開（§7.7）。
入力方式: **貼り付け方式（既定）**＝項目をクリップボードへ置き Ctrl+V（IME・特殊
文字に強い）。Tab/Enterはキー送出。1文字送出も設定可だが既定は貼付。
中断: 入力中Escで即中断（途中まで残し残りを止め「i/N項目で中断」）。フォーカスが
書込先から外れたら安全中断。各送出間に微小ウェイト（既定30ms）。

## 7.6 表モード

収集パネルの「表として取り込み」or 収集中に Shift＋ドラッグ。表をドラッグ→Ctrl+C
で多くの表はタブ区切り＋改行が得られる→TSVを1項目（種別=表TSV）として格納。
流し込み時、表項目はクリップボードにTSVを置き Ctrl+V 一回で展開（Excel=行列、
Word表=対応セル）。コピー時にTSV化されない表は通常項目に分けて拾う運用。

## 7.7 クリップボード退避・復元

収集開始時に現内容を退避→収集(Ctrl+C)・流し込み(Ctrl+V)で作業利用→完了/中断/
終了で退避内容を復元（失敗時も空にしない努力）。「ツールを使ったら元のコピーが
消えた」を起こさない。

## 7.8 潔白性・設定・チェック・連携

潔白性: 読み元 read-only。文字の中身は履歴的に保存しない（バッファは作業用で
永続しない）。
設定: hotkey_collect(Ctrl+Alt+C) / hotkey_flush(Ctrl+Alt+V) / flush_separator
(Tab) / input_method(paste) / key_interval_ms(30) / copy_poll_ms(300) /
table_modifier(Shift) / confirm_before_flush(true) / abort_on_focus_loss(true) /
restore_clipboard(true) / recording_conflict(pause) / keep_buffer_after_flush
(true)。
チェック: [ ] 収集（自動Ctrl+C・ポーリング・自プロセス/極小除外）[ ] パネル（並べ
替え/削除/編集）[ ] 確認プレビュー（件数/順番/対象窓）[ ] Tab送り（区切り切替）
[ ] 貼付方式（IME耐性）[ ] Esc中断・フォーカス喪失中断 [ ] 表モード（TSV一括）
[ ] クリップボード退避復元 [ ] 読み元read-only。
連携: 履歴/付箋/検索→転記バッファ（③）。整形・マッピングは③第1波。コマンド
供給: 収集開始/流し込み/収集パネル。

# 8. クリップボード履歴

═══════════════════════════════════════════════════════════════════

コピーした内容を遡って再利用。転記で整備したクリップボード基盤を再利用する。

## 8.1 目的・確定仕様

目的: 直前1個しか残らないコピーを遡れるようにし、定型を固定し、軽く整形して
貼れる。
確定: クリップボード更新を購読（ポーリング併用・テキスト主体）/ 既定50件リング
バッファ / 永続化しない（揮発・再起動で消える）/ ホットキーで履歴パネル→選択で
クリップボードへ復帰（任意で自動貼付）/ ピン留め（定型を上部固定）/ 機微情報
（パスワード様・除外アプリ由来）は残さない / 軽い整形（和暦/全半角/カンマ等）/
複数選択→転記バッファへ。

## 8.2 設計思想

- 転記のクリップボード退避・設定・復元・監視を流用（「監視して貯める」「選んで
  戻す」を足すだけ）。
- 履歴は揮発する作業メモ（永続化しないことが機微の長期残留を防ぐ最大の安全策）。
- 潔白性最優先（税務の値が残る前提）: 永続化しない・パスワード様/除外アプリは
  最初から記録しない・ログに内容を書かない。

## 8.3 監視

クリップボード更新通知を購読＋低頻度ポーリング（既定500ms）。対象はテキスト
（CF_UNICODETEXT・画像等は将来）。自己起因（復帰での設定）・転記由来の変更は履歴
汚染しない（黒猫§B13.4標識流用）。連続重複・空は追加しない。既存同一は先頭へ
繰り上げ。トレイ/パネルから監視一時停止（機微作業中に履歴を作らない）。

## 8.4 履歴パネル・ピン・整形

パネル: ピン領域→履歴領域（新しい順）。検索（インクリメンタル部分一致）・↑↓・
数字即選択・複数選択。各行=プレビュー・由来アプリ・時刻。
復帰と自動貼付: 決定→クリップボードへ再設定（復帰）。auto_paste 有効なら記憶した
直前ウィンドウへ Ctrl+V（送出キーは記録除外）。
ピン: よく使う定型（署名・定型句）を上部固定（リング溢れで消えない）。順序変更・
ラベル付け可。セッション内保持（永続は③第1波で任意）。
整形（任意適用・流用エンジン）: 全角半角 / カンマ除去・付与 / 和暦西暦 / トリム /
数字のみ抽出。決定的・誤変換しそうは適用前プレビュー。税務文脈でカンマ・和暦・
数字抽出を主役に。

## 8.5 機微情報の除外（潔白性の核）

パスワード様: パスワードフィールド由来（UIA判定可なら）は残さない。形式推測は
弱い補助に留め過剰除外しない。除外アプリ: 設定で由来アプリ指定（パスワード管理
ソフト等）。残さない徹底: ログに内容書かない・永続化しない（再起動で全消去）・
「履歴を今すぐ全消去」をパネルとトレイに。

## 8.6 データモデル・設定・エッジ・チェック・連携

ストア: 履歴=固定長リング（既定50・先頭最新）／ピン=別リスト（溢れ無関係）。
項目: value/preview/source_app/created_at/label/pinned。
設定（名前空間 `clipboard.` 配下・①§B10。`hotkey_panel`は検索の同名設定と
名前空間で分離され衝突しない）: hotkey_panel(Ctrl+Alt+X) / history_size(50) / poll_ms(500) /
auto_paste(false) / exclude_apps([]) / exclude_password_fields(true) /
dedup_consecutive(true) / formatting_enabled(true) / keep_pins_only_persist
(false) / watch_enabled(true)。
エッジ: 取得失敗→スキップ / 巨大テキスト（>1MB）→先頭のみ保持 / 自動貼付先変化
→中止 / 整形不能→元のまま復帰 / リスナ切れ→再登録・不可ならポーリングのみ。
チェック: [ ] 購読＋ポーリング・自己/転記由来除外 [ ] 連続重複/空除外・既存繰上
[ ] 監視一時停止 [ ] リング＋ピン [ ] パネル（検索/即選択/複数選択）[ ] 復帰＋
任意自動貼付（記録除外）[ ] ピン（順序/ラベル）[ ] 整形＋プレビュー [ ] パスワード
/除外アプリ残さない・永続化しない・即時全消去。
連携: 履歴/ピン→転記バッファ（複数選択・③）、履歴→付箋（③）。整形エンジンは
転記と共有（③第1波）。コマンド供給: 履歴パネル/全消去/監視停止再開。

# 9. ManualRecorder（操作記録→マニュアル生成）

═══════════════════════════════════════════════════════════════════

記録ON中のクリック/ドラッグ/入力をスクリーンショット付きで記録し、記録OFF時に
手順書HTML＋TSVを自動生成する。UIA前提・多段フォールバック・混在アプリ対応。
（本機能は規模が大きく、設計の核を凝縮。元 v1.1 設計書が詳細版）

## 9.1 目的・優先順位

初回作業の手順書を2回目以降に迷わず再現できる粒度で作るのに時間がかかる問題を、
操作記録から手順書下書きを自動生成して圧縮する。
優先順位: ①潔白性（キー内容を残さない・通信ゼロ・記録物の透明性）②堅牢性（1操作
失敗で常駐を止めない）③手順文の可読性 ④見栄え。

## 9.2 ホットキー・状態

F9=記録開始/終了（保存） F10=一時停止/再開 F8=直前手順へ注釈 F7=メモ挿入
Ctrl+F9=緊急破棄。状態 IDLE/RECORDING/PAUSED（インジケータ色 灰/赤/黄）。
状態遷移時、保留クリック・タイピングバーストは確定/破棄して持ち越さない。

## 9.3 スレッドとスクショ同期撮影

フックは「スクショ撮影（同期）＋投入」までで即return、重い処理（UIA最大700ms）は
ワーカー。**撮影だけフックスレッドで同期実行**＝クリック後に画面遷移する前を確実に
捉える。整形（マーカー描画・拡大切出し・JPEG化）はワーカー。

## 9.4 UIA多段フォールバック（中核）

混在アプリ（Win32/ブラウザ内/特殊FW）すべてで手順を成立させる心臓部。単一
パイプラインで上から試し取れた段で採用:

|段 |条件                    |手順文の形           |
|--|----------------------|----------------|
|T1|Name有・Type有           |「『登録』ボタン」       |
|T2|Name空・近傍ラベル発見         |「入力欄（『整理番号』付近）」 |
|T3|AutomationId/ClassName|「入力欄（id: txtNo）」|
|T4|名前情報皆無                |「ボタン（画面の赤丸位置）」  |
|T0|取得不能/例外/タイムアウト        |「赤丸の位置」         |

どの段でもウィンドウ名＋スクショ赤丸は併記＝T0でも再現可能。アプリ種別
（A:Win32/Office, B:ブラウザ, C:特殊FW）は探索順の最適化に使うのみ（判定外れても
破綻しない）。近傍ラベル探索は左・上優先・200px以内最近傍。UIA1問い合わせに上限
700ms（超でT0降格）。

## 9.5 プライバシー（変更不可の中核）

- **文字入力内容の非記録**: 内容を保持・ログ・スクショ以外で残さない。「入力という
  行為」のみ記録。
- **IME対応（堅い割り切り）**: Enter/Space/Tabをバースト区切りに使わず、無入力
  時間（既定2.5s）・マウス操作・ウィンドウ変更・状態遷移・ショートカット発生で
  区切る。IME状態取得に依存せず堅牢。日本語の変換確定Enterで入力が分断されない。
- **ショートカットの意味付き記録**: 修飾付き（Ctrl+S等）は「保存する」と意味で
  記録（文字の中身は残さない）。標準辞書（Ctrl+S/C/V/X/Z/Y/A/F/H/P/N/O/W、F2、
  F5、Delete、Alt+Tab等）。連打は回数集約（「元に戻す（×3）」）。辞書外は「Ctrl+G
  を押す」。自ツールホットキーは除外。
- スクショ: F10一時停止中は撮らない。配布前マスキング確認をHTMLフッタに常時印字。
- 遅延書き込み: 記録中はメモリのみ（中間ファイルも作らない）。保存物はHTML/TSV
  のみ。Ctrl+F9で即解放。
- メモリ上限: 警告300MB・上限600MBで劣化退避（拡大画像省略→品質低下）でも記録
  継続（止めるより画質を犠牲に）。
- UAC/セキュアデスクトップ: 保護画面中の欠落を自動注記で可視化。
- 外部通信ゼロ。

## 9.6 操作解釈

クリック種別: 距離≥18px かつ押下≥120ms=ドラッグ（手ブレ抑止）。ダブルクリックは
同一ボタン・0.45s・8px・**同一UIA要素**・未注釈をすべて満たすときのみ統合（意図的
連打を潰さない）。修飾＋クリック（Shift/Ctrl+クリック）は「Shiftを押しながら…」と
記録（Excel範囲選択等）。

## 9.7 注釈（F8）・メモ（F7）

F8: 直前の注釈可能手順へ「入力内容（ざっくり）」「判断根拠」の2欄をその場登録。
判断根拠は手順文に混ぜず独立欄（「なぜその入力か」を際立たせる）。対象手順の番号・
手順文を明示し、対象を数手順前へずらせる（取り違え防止）。手書きテキストなので
キー内容非記録と両立。F7: 任意位置に注意書きを手順挿入（スクショなし）。

## 9.8 出力

HTML（本命・単一ファイル）: 画像base64埋め込み・**JavaScript一切なし**（静的・
可搬・安全）。手順カード（番号バッジ＋手順文＋時刻＋注釈タグ＋スクショ赤丸＋拡大）。
印刷でカード分断しない（break-inside:avoid）。フッタに「自動生成下書き・要マスキング
確認・キー内容未記録」を常時印字。
TSV（取り込み用・新規スキーマ）: UTF-8 BOMなし・1行目バージョン宣言
（#MANUALREC_TSV v1.1）・列=手順番号/時刻/種別/アプリ/ウィンドウ/取得段/部品/修飾/
キー/操作文/入力内容/判断根拠。1行1手順・「操作文（生成済み）」と「部品/段/修飾/
キー（素材）」を両方持ち取り込み側が再生成してもしなくても対応。

## 9.9 手順データモデル

no/time/type(click|drag|input|shortcut|memo|note)/window(正規化)/window_raw/exe/
app_kind/tier/elem/btn/modifiers/double/shot/closeup/key_count/sc_keys/sc_meaning/
sc_repeat/text/note_text/note_content/note_basis（＋内部判定用 _press_pos/_press_t/
_elem_key）。ウィンドウタイトル正規化: ブラウザのプロファイル/ブラウザ名サフィックス
除去・通知カウンタ除去・60字短縮（手順の冗長化防止）。

## 9.10 耐障害・死活監視

1操作の失敗で常駐を止めない（UIA失敗→T0降格・スクショ失敗→画像なしで継続・
ワーカー例外→当該イベント破棄し継続）。フック死活監視（ハートビート・記録中なのに
60秒イベント無→遮断の可能性を通知・再登録試行）。UAC兆候→自動注記（§9.5）。

## 9.11 依存・環境・設定（抜粋）

依存: pynput・Pillow 必須、uiautomation(+comtypes) 強く推奨（無いとT0固定）、
tkinter 必須（embeddable不可）。修飾キー状態は GetKeyState/GetAsyncKeyState 推奨。
設定（抜粋）: max_width(1366)/jpeg_quality(72)/typing_idle_sec(2.5)/dblclick(0.45/8)
/drag(18/120ms)/uia_timeout_ms(700)/show_key_count(false)/mem_warn/cap(300/600)/
heartbeat/idle_alert(5/60)/shortcut_dict/hotkeys。

## 9.12 制約

VDI/リモート画面は非対象（UIAが実体に届かず全T0）＝対象はローカル実行（確認済み）。
Java製C種別はUIAブリッジ無でT3/T4止まり。UAC中の操作内容そのものは復元不可。
独自ショートカットは汎用表現（手で意味補完）。
