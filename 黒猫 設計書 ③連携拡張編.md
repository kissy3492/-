# 黒猫 設計書 ③ 連携拡張編 v1.0

黒猫の全4編のうち第3編。(A) 機能間連携の一望と規約、(B) 第1波拡張（各機能内の
8拡張）、(C) コマンドマクロ（第2波の目玉）、(D) 拡張ロードマップ全体を収める。
基盤編①・機能編②を前提とし、共通規約（①第D部）に従う。

- 前提: ①基盤編 v1.0 ／ ②機能編 v1.0
- 不変原則: ①§D1（外部通信ゼロ/機微非保持/ローカル完結/疎結合/モード調停 等）
- 本編の役割: 機能を繋ぎ、育てる「地図」。新ロジックではなく組み合わせと発展の
  指針を定める。

> **★実物向けの読み替え（軽微）**: 黒猫は既存のランチャー（別添「追加実装ガイド」
> 参照）。本編の連携・拡張は、②機能編の機能が**複数そろってから**使う。最初の
> 数機能を足す段階では不要。連携IF・拡張ロードマップは実装手段に依存しないので
> そのまま有効。第3波（税務計算・OCR・M365）はポリシー審査が先（§D3）。

# 第A部　機能間連携の一望と規約

═══════════════════════════════════════════════════════════════════

## A1. 連携の大原則（①§B2,§B5.2 準拠）

- 機能同士は**直接依存しない**。連携は必ずシェルのモジュール間IF経由（疎結合）。
- 受け渡すのは**最小限のデータ**（テキスト・パス・画像参照）に限る。
- 連携が対話モードやキー送出を伴う場合、**モード調停**（①§B7）に従う。
- 連携でも**機微を残さない**原則は不変（①§B15.4）。

## A3. 連携の技術規約

### A3.1 コマンド供給（全機能→パレット）

register時にコマンドを申告（①§C6・本編で全機能が利用）。機能起動・機能内
アクション・動的コマンドがパレットに集約。新機能もコマンド申告だけで載る。

### A3.2 テキスト受け渡しバス（履歴・付箋・転記・検索）

テキスト系連携を共通規約に統一: 1件＝1テキスト（複数は順序付きリスト）/ 宛先は
「クリップボードへ」「転記バッファへ」「付箋へ」/ 受け手が非アクティブなら
モード調停に従い起動可否確認 / 受け渡したテキストは実行に使うのみ・中継ログに
残さない。
対応: 履歴→転記（②§8）・付箋→転記/クリップボード（②§5）・検索→転記/
クリップボード（②§4）・履歴→付箋（②§5,§8）。

### A3.3 ウィンドウ操作の委譲（呼出・作業状態→配置）

「枠へ並べる」は配置に委譲（対象ウィンドウ＋枠 または 実矩形 を渡すだけ。占有・
topmost・矩形計算は配置が担う）。委譲はモード調停に従う。

### A3.4 起動・前面化の委譲（作業状態・検索→呼出）

「アプリ起動／既存窓前面化」は呼び出しの解決ロジックに委譲。PowerShell非依存の
起動方針を共有。

### A3.5 画像受け渡し（領域→履歴）

領域キャプチャの画像はクリップボード（画像形式）経由で履歴へ。画像は揮発前提・
履歴側も長期保持しない。

### A3.6 記録除外の徹底（全機能→記録）

各機能が送出するキー（転記のCtrl+C/V/Tab、履歴/付箋/領域の自動操作）に「自ツール
起因」標識を付け、記録の手順化対象から除外（①§B13.4）。連携が記録を汚さない
ことをシェルが保証。

## B1.【★厚】転記の整形（→項目転記）

単純コピーだった転記に**任意の整形を挟む**。税務の数値・日付・番号整形に直結。

- **整形エンジンは履歴（②§8.4）と共通の1モジュール**にまとめ双方から呼ぶ（①§B12
  共通ユーティリティ層に置く・二重実装しない）。決定的・可逆志向・誤変換はプレビ
  ュー必須。
- 整形メニュー（税務重点）: カンマ除去/付与・全角半角・和暦西暦・ゼロ埋め（桁指定）
  ・数字のみ抽出・トリム・区切り変換。
- 適用3レベル: ①項目ごと（収集パネルで各項目に指定。氏名はトリム・金額はカンマ
  除去等混在）②一括（全項目に同じ整形）③整形プリセット（「申告書入力用」等・
  複数整形の組を名前付き保存・ワンタッチ）。
- 適用は**流し込み直前**・元の収集値は壊さない（派生）。確認プレビュー（②§7.5）に
  整形後の値を表示・整形前後を見比べ・和暦変換等で変換不可は元のまま流す。
- プリセット: name/rules（順序付き整形ステップ）/scope。ローカル永続（ルールのみ・
  機微なし）。パレットへ「整形プリセットを適用」供給。
- 潔白性: 整形対象の値（機微）は保存・ログしない。永続はルール定義のみ。
- チェック: [ ] エンジン履歴共有 [ ] 税務整形メニュー [ ] 3レベル適用 [ ] 直前
  適用・プレビュー整形後表示・元値保持 [ ] プリセット永続（ルールのみ）[ ] 値非保存。

## B2.【★厚】作業状態の複数スロット（→作業状態）

1スロット上書きを**事案ごとに名前付き保存**へ。複数事案を抱える調査に直結。

- スロットモデル拡張: slot_id/name（事案名等ラベル）/saved_at/monitor_profile/
  entries。スロット数上限ゆるめ・各独立。
- 保存: 「新規スロット」or「既存に上書き」（上書き時1世代バックアップ・②§3.8）。
- 復元: スロット一覧（名前・日時・件数）から選択・あいまい検索で絞る。パレットへ
  「保存（新規/上書き）」「復元（一覧・動的候補）」供給。
- 機微配慮: 保存内容は従来どおりパス・窓情報・配置のみ（中身非保存・②§3.7）。
  複数化で原則不変。スロット名（事案名）/パスは機微になりうる→ローカル限定・
  一覧の「事案名を伏せ字表示」目隠し（来客時）・「全/特定スロット消去」。
- 非破壊: 復元は既存窓を閉じない。切替時は「現在を別スロットに保存してから復元」を
  案内。「復元前に現在の窓を一括最小化/退避」片付けオプション（任意）。
- 第2波布石: マクロが特定スロット復元を呼べるよう slot_id で指定可能に。
- チェック: [ ] 名前付き複数 [ ] 新規/上書き・一覧復元・検索 [ ] パレット動的供給
  [ ] 中身非保存維持・名前伏せ字・即時消去 [ ] 非破壊・片付け [ ] slot_id指定。

## B3.【★厚】転記マッピング保存（→項目転記）

毎回手で範囲指定していたのを「この転記セット」を**名前付き保存して再現**。

- **重要: 実データ（収集値）は保存しない。保存は「構成」だけ**。
- マッピング: name/slots（項目スロット定義）/flush_separator/formatting_preset（任意）。
- 項目スロット: order/label（「整理番号」等）/formatting（この項目の整形・B1）/
  required（未収集なら警告）。
- 使い方: マッピングを選ぶ→収集パネルに空のラベル付きスロットが並ぶ→読み元で各値を
  ドラッグ収集→対応スロットへ順に入る（ラベルで対応明確）→定義済みの順・区切り・
  整形で流し込み。
- 確認強化: ラベルと収集済み値の対応をパネル表示・required未収集は流し込み前警告。
- 潔白性: マッピング定義（構成・ラベル・整形）のみ永続・**収集値は保存しない**・
  ラベルが事案示唆しうる→ローカル限定・即時消去。
- チェック: [ ] 構成のみ保存・値非保存 [ ] 項目スロット（順/ラベル/整形/必須）
  [ ] ラベル付き空スロットへ収集→定義通り流し込み [ ] 対応表示・必須警告 [ ] パレ
  ット動的・整形プリセット参照。

## B4.【要点】履歴のピン永続化（→履歴）

履歴は揮発原則だが**ピンだけ**永続化オプション（定型文をセッション跨ぎ保持）。
対象はピンのみ（履歴本文は揮発維持）・既定オフ（keep_pins_only_persist）・ピン本文に
機微を入れない注記・ローカル限定・「永続ピンを全消去」・履歴本文とは別ファイル管理。
チェック: [ ] ピンのみ永続 [ ] 既定オフ [ ] 全消去 [ ] 本文揮発維持。

## B5.【要点】付箋のアラーム（→付箋）

付箋に alarm_at（指定時刻）を任意設定→到来で前面化＋点滅（黒猫が能動通知する数少ない
例）。繰り返し最小サポート。常駐の軽量タイマー（①§B14共有）で実現・重い監視を持た
ない。アラーム時刻はローカルのみ。
チェック: [ ] alarm_at [ ] 到来で前面化＋点滅 [ ] 軽量タイマー共有 [ ] 繰り返し最小。

## B6.【要点】領域の注釈描画（→領域）

切り出し画像に矩形・矢印・フリーライン・文字（色数色）。描画は別レイヤ（元画像保持・
消せる）。保存/コピー時に「描画込み/元のみ」選択。揮発原則維持（再起動で消える・
描画も永続化しない）。
チェック: [ ] 矩形/矢印/線/文字 [ ] 別レイヤ消去可 [ ] 保存時に描画込み/元のみ
[ ] 揮発維持。

## B7.【要点】ファイル検索の絞り込み強化（→検索）

お気に入り固定（よく開くファイルを無入力時最上段）/ 簡易構文拡充（folder: since:
until: 等）/ 検索プロファイル（対象フォルダ＋絞り込みの組を名前付き保存）。索引は
メタのみ・中身読まない維持・お気に入り/プロファイルはパス/条件のみ永続・クエリ
非保存。
チェック: [ ] お気に入り [ ] 構文拡充 [ ] プロファイル [ ] メタのみ・クエリ非保存。

## B8.【要点】記録の注釈追記モード（→記録）

F8注釈を上書きでなく**追記（履歴化）**できるオプション（入力内容・判断根拠を複数回
に分けて足す）。既定は上書き（後方互換）。出力は追記注釈を箇条で並べる。キー内容
非記録は不変（注釈は手書きテキストのみ）。
チェック: [ ] 再F8で追記 [ ] 既定上書き [ ] 出力で箇条表示 [ ] キー内容非記録維持。

## B9. 横断事項（8拡張共通）

- **後方互換**: 全拡張は既定オフ/従来動作を保ち有効化で効く。
- **永続化が増える項目の潔白性原則**: 「構成・定義・ラベルは永続してよい。実データ
  （値・本文・中身）は機能の既存方針に従う」。複数化・テンプレ化で機微保存を増やさ
  ない。
- **コマンド供給（第2波布石）**: 各拡張が追加するアクション（整形プリセット適用・
  スロット復元・マッピング収集開始・検索プロファイル等）をパレットへ供給＝マクロの
  素材。
- 親への影響: 共通整形エンジン（B1）を①§B12共通ユーティリティ層に置く小追加のみ
  （既存動作を変えない）。

# 第C部　コマンドマクロ（第2波の目玉）

═══════════════════════════════════════════════════════════════════

複数コマンドを連結し1キー/1呼び出しで順次実行。パレットがコマンドを集約した構造
（①§C6）と第1波で各アクションがコマンド化された前提（§B9）の上に乗る。黒猫を
「部品の集合」から「段取りを回す道具」へ進化させる。

## C1. 目的・確定仕様

目的: 事案着手・転記準備など「決まった手順で複数機能を順に使う段取り」を1アクション
に圧縮（§A4のシナリオを1キー化）。
確定: パレットのコマンドを順序付きで束ねる「台本」/ 記録ではなく組み立て（選んで
並べる）/ ステップ間は逐次（窓出現等は待ち合わせ）/ ステップに引数束縛（スロットID・
枠・マッピング名等）/ 失敗時は既定中断＋レポート（ステップ単位で継続指定可）/
起動はパレット/専用ホットキー/トレイ / 実行前プレビュー・途中Esc中断・モード調停・
機微非保持。

## C2. 設計思想

- **マクロ＝コマンドの台本**: 新実行系ではなく既存コマンドを順に呼ぶ。各ステップは
  パレットが知るコマンド（①§C6）そのもの。「どのコマンドを・どんな引数で・どの順で・
  どう待って」呼ぶかを記述するだけ。
- **記録式ではなく組み立て式**: 操作を録って再生するRPA的手法を取らない（座標依存・
  壊れやすい・記録に機微混入）。意味あるコマンドを選んで並べる＝台本は読めて壊れにく
  く機微を含まない。
- 安全第一: ステップ実行もモード調停（①§B7）に従う（逐次なので自然に守られる）。
  「閉じる・消す」系を含むなら実行前に明示。
- 潔白性: マクロ定義に保存するのは「コマンドid＋非機微引数（スロットID・枠・プリ
  セット名等）」のみ。値（氏名・金額・本文）は焼き込まない。

## C3. データモデル

マクロ: macro_id/name/hotkey/steps/on_error_default。
ステップ: order/command_id/args（非機微引数）/wait（待ち合わせ）/on_error/optional。
扱えるのはパレット供給済み全コマンド（機能起動・アクション・第1波で増えたもの）。
enabled_when偽は実行時スキップ＋記録。

## C4. 構築UI（組み立て式）

新規作成→名前→コマンドを検索追加（パレットと同じあいまい検索）→順に並べ各ステップ
に引数・待ち合わせ・失敗時方針→並べ替え/削除/複製→保存。引数が選択肢を持つコマンド
（スロット復元・枠・マッピング選択等）は動的候補（①§C6.3 arg_provider）から選ぶ
（手打ち取り違え防止）。テスト実行（本番と同じプレビュー・中断つき）。

## C5. 実行制御

- 逐次（既定）: order順に1つずつ・前完了後に次・同時並行しない（モード調停・安全）。
- 引数束縛: 非機微パラメータのみ（スロットID・枠名・整形プリセット名等）。値は束縛
  しない。
- 待ち合わせ: none / window_appear（窓出現待ち・タイムアウト・②§2.2流用）/
  fixed_delay（一定時間・最終手段）/ settle（完了通知待ち）。タイムアウトは失敗扱い・
  無限待ちしない。
- ステップ間: 前ステップのモードが解放されてから次を起動（モード調停が保証）。

## C6. 失敗時の方針（暴走させない）

- 既定: 失敗（例外・タイムアウト・実行不可）で**そこで中断**＋レポート。中途半端でも
  勝手に巻き戻し/閉じない（非破壊）。
- ステップ単位: on_error=continue で次へ進む（補助ステップ向け）。optional=true は失敗
  を成功扱い。enabled_when偽はスキップ記録し継続。
- 危険ステップ: 「閉じる・消す・全消去・上書き」系を含むマクロは実行前プレビューで
  警告色明示。

## C7. 実行UI

- 実行前プレビュー: 実行せずステップ列表示（コマンド名・引数（事案名等）・待ち合わせ・
  失敗時方針・破壊的ステップは警告色）。「実行/キャンセル」。設定で確認省略も可（信頼
  マクロ用・既定は確認あり）。
- 進捗・中断: 現在ステップと進捗（3/6）をインジケータ表示。Escでいつでも中断（実行済み
  は保持・残り停止・非破壊）。
- 実行レポート: 各ステップ結果（成功/スキップ/失敗/継続）一覧・失敗は理由＋手戻し導線。
  保存・ログしない。

## C8. 代表マクロ例（§A4の1キー化）

|マクロ    |ステップ概要                                  |
|-------|----------------------------------------|
|○○事案を開く|作業状態を復元(事案スロット)→待ち(窓出現)→配置(左右)→付箋(本日の確認)|
|転記準備   |領域キャプチャ(参照表)→マッピングで収集開始(基本情報セット)        |
|別資料セット |検索で資料A→待ち→呼出で資料B前面化→配置(左右)              |
|退庁前    |作業状態を保存(上書き)→付箋を全退避→領域キャプチャを全消去         |
|記録開始セット|配置(作業画面)→記録開始                           |


> 既存コマンドの組み合わせのみ。新ロジックを足さず作れる。

## C9. 潔白性・データ

マクロ定義に保存するのは「コマンドid＋非機微引数」のみ・**値は焼き込まない**。
例外的にファイルパスを引数に持つ場合はパスが機微になりうる→ローカル限定・「マクロ
消去」・一覧の事案名伏せ字（B2と同方針）。実行時に各コマンドが扱う機微は各機能の
既存方針に従い保存・ログしない。実行レポート・進捗は保存しない。外部送信ゼロ。

## C10. 既存機能との関係

マクロはコマンドを呼ぶだけ（独自のウィンドウ操作・フックを持たない）。ステップ実行は
モード調停に従い対話モードは逐次で同時1つ維持。呼ぶコマンドの自動操作キーは記録除外
（①§B13.4）。マクロもコマンドとしてパレット登録（マクロ→マクロは既定1段に制限＝
循環防止）。第1波で増えたアクションを主要部品に使う（§B9）。

## C11. 設定・エッジ・チェック

設定: confirm_before_run(true) / default_on_error(abort) / wait_timeout_ms(8000) /
fixed_delay_ms(600) / allow_nested_macro(false) / hide_case_names(false) /
macro_hotkeys（マクロごと任意）。
エッジ: id不明→失敗・理由表示（定義陳腐化）/ タイムアウト→既定中断/指定継続 /
enabled_when偽→スキップ継続 / 破壊的→プレビュー警告 / 循環→多段制限で防止 / 実行中
構成変化→各コマンドの既存耐性に委ね失敗はレポート / 中断→実行済み保持・残り停止。
チェック: [ ] マクロ/ステップモデル [ ] 組み立てビルダ・引数は動的候補・テスト実行
[ ] 逐次・モード解放待ち [ ] 引数は非機微のみ [ ] 待ち合わせ（窓出現/遅延/完了・
タイムアウト）[ ] 失敗時方針（既定中断・継続/optional/スキップ）[ ] 破壊的警告
[ ] 実行前プレビュー・Esc中断・非破壊 [ ] レポート保存しない [ ] 定義に値焼き込ま
ない・ローカル限定・消去 [ ] モード調停・記録除外 [ ] マクロ供給・ネスト既定オフ。
